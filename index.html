Breadth-First Search (BFS)

Explores all nodes level-by-level. Guarantees the shortest hop-path from entry to exit. O(V+E) time complexity.

Greedy Best-First Search (GBFS)

Always expands the node closest to the goal using Manhattan Distance heuristic h(n). Fast but not guaranteed optimal — may get trapped.

Depth-First Search (DFS)

Explores one path deeply before backtracking. Does not guarantee shortest path. Useful for exploring all possible routes.

Uniform Cost Search (UCS)

Expands the node with the lowest cumulative path cost. Optimal for weighted graphs. No heuristic — guaranteed shortest weighted path.

A* Search Algorithm

Combines g(n) actual cost + h(n) heuristic. f(n) = g(n) + h(n). Optimal AND efficient — the gold standard for pathfinding.

04 — System Design

Four-Stage Processing
Pipeline

The real-time system integrates graph construction, pathfinding, and optimization into a unified four-stage pipeline that operates continuously during events.

1

Graph Construction & Update

Build or refresh the weighted graph using latest sensor data. Recalculate all dynamic edge weights based on current congestion. Disable edges/nodes flagged as hazardous or blocked.

↓
2

Pathfinding — BFS & Bidirectional Search

Execute BFS for rapid proximity-based exit discovery. Run Bidirectional Search for optimal route computation on the full graph. Produce shortest-path candidates for each occupied zone.

↓
3

Flow Optimization — Simulated Annealing

Apply SA to the current routing assignment. Iteratively perturb crowd distribution across exits. Minimize global congestion cost function. Output optimized flow allocation per exit/corridor.

↓
4

Guidance Generation

Translate optimized routes into actionable guidance: LED signage updates, PA announcements, mobile app push notifications, and security personnel instructions.

05 — Results & Advantages

Performance Outcomes &
System Advantages

⚡

Faster Route Discovery

BFS completes in microseconds for 150-node graphs, enabling routes for thousands simultaneously. Bidirectional Search reduces search space exponentially.

📊

Congestion Reduction

Dynamic edge weighting ensures congested corridors are automatically disfavored. SA prevents the "popular exit" problem by explicitly distributing load.

🛡️

Improved Safety

Crowd densities maintained below dangerous thresholds in real time. Emergency rerouting completed within seconds of a hazard alert.

🔄

Real-Time Adaptability

Unlike static evacuation plans, the system adapts continuously. Blocked corridors trigger immediate graph updates and new path computation.

📈

Scalability

The same framework applies to airports, shopping malls, pilgrimage sites, and city-wide events — simply by loading a different venue map.

🔗

IoT Integration

Seamlessly integrates with sensor networks, thermal cameras, Bluetooth/Wi-Fi density analysis, and computer vision systems.

Known Limitations & Challenges

Every system has constraints. These are the key challenges to address in future work.

Real-Time Data Dependency

Sensor failures or signal interference introduce stale data, leading to suboptimal routing recommendations.

Computational Overhead

SA has no guaranteed convergence time. For extremely complex venues, optimization may lag real-world conditions.

Human Compliance

People may ignore digital signage or take familiar paths regardless of system guidance. Human marshals remain essential.

Infrastructure Cost

Full deployment requires sensor networks, cameras, centralized computing, and digital signage — potentially cost-prohibitive for older venues.

Key Takeaways

01Graph-based venue modeling provides a rigorous mathematical foundation for crowd management — turning a physical space into a solvable computational problem.
02BFS guarantees fast, correct evacuation route discovery with O(V+E) complexity — completing in microseconds for typical stadium graphs.
03Bidirectional Search delivers exponential speedups — exploring ~512× fewer nodes on a branching factor 4, depth 10 graph.
04Simulated Annealing provides dynamic, globally-aware congestion optimization — preventing the "popular exit" problem across all evacuation routes.
05Combined, these techniques form a scalable, adaptable, and genuinely life-saving system for any large-scale event or venue worldwide.
